الكائنات Objects

كل شي في هذه الحياة هو عبارة عن Object (كائن / شئ).

أنظر لأي شي حولك ستجد أنه Object يحمل عدد من الخصائص (Properties) و الأفعال (Methods).

لنأخذ على سبيل المثال الجوال:

  • خصائص الجوال: الشركة المصنعة, الموديل, السعر, اللون, الخ …

  • أفعال الجوال : الإتصال, إرسال الرسائل, تشغيل الموسيقى, الخ …

تلاحظ أن كل هذه البيانات تحت مسمى شئ واحد وهو phone.

إذا نستطيع القول بأن كل ما حولنا هو Objects تحمل خصائص (Properties) وأفعال (Methods \ Actions).

الـ Objects في لغات البرمجة

ينطبق على الـ Objects نفس الفكرة في لغات البرمجة فعند تعاملك مع الجافاسكربت ستجد الـ objects في كل مكان فكل مافي لغة جافاسكربت من مصفوفات ودوال وأنواع البيانات وغيرها إما Objects او بالإمكان التعامل معها كـ Objects لهذا تجد أنه يطلق على الجافاسكربت بـ Object Oriented Programming او OOP.

فالـ OOP هو نمط برمجة يتم فيه الإعتماد على تقسيم البرنامج إلى وحدات تسمى Objects كل وحدة تتكون من حزمة من البيانات يتم لاحقاً التعامل مع هذه الوحدات بالطريقة المناسبة حسب المتطلبات.

في لغات البرمجة الـ Object يتكون من شيئين:

  • متغيرات وتسمى بالـ (Properties)

  • الدوال والتي تسمى بالـ (Methods)

لنقم بكتابة الجوال بلغة الجافاسكربت كـ object

  • الخطوة الأولى سأقوم بتعريف الـ Object كالتالي:
var phone = { };

الأن سأبدأ بكتابة الـ الخصائص (Properties) كالتالي:

var phone = {

    company : "Apple"

};

لاحظ كيف قمت بكتابته كما موضح بالصورة التالية:

لنكمل كتابة بقية الخصائص الأخرى ونضع بين كل خاصية واخرى فاصلة كالتالي:

var phone = {

   company : "Apple",
   name : "iphoneX",
   price : 1200,
   color: "back"

};

رائع! إنتهينا من كتابة الـ Properties

الأن لنقم بكتابة الـ Methods:
كما ذكرنا أعلاة الـ methods هي عبارة عن دوال وكما قمنا بالشرح في الدرس السابق عن طرق تعريف الدوال وإستخدامها سنقوم بمثل ذلك في هذا الدرس كالتالي: 

var phone = {

    /*Properties*/
    company : "Apple",
    name : "iphoneX",
    price : 1200,
    color: "back", // لا تنسى أضافة الفاصلة بين كل عنصر

    /*Methods*/
    call : function(){
       return ("calling ...");
    },
    sms :  function(){
        return ("sending sms ...");
    }

};

سنكتفي بإثنتين من الـ methods.

الوصول لعناصر الـ Object

نستطيع الوصول لعناصر الـ object بطريقتين كالتالي:

لنقم بتجربة الطريقة الأولى بكتابة إسم الـ object ومن ثم نقطة ومن ثم إسم الخاصية كالتالي:

var phone = {

    /*Properties*/
    company : "Apple",
    name : "iphoneX",
    price : 1200,
    color: "back", // لا تنسى أضافة الفاصلة بين كل عنصر

    /*Methods*/
    call : function(){
       return ("calling ...");
    },
    sms :  function(){
        return ("sending sms ...");
    }

};


console.log( phone.company );
console.log( phone.name );
console.log( phone.price );

console.log( phone.call() );

var phone = {

    /*Properties*/

    name : "iphoneX",

ماهي فائدة الـ Objects

بإمكانك إعتبار الـ Object بأنه يقوم بجعل المتغيرات والدوال التي تربطها علاقة مشتركة في مجموعة كحزمة يسهل لك الوصول إليها والتعامل معها بكل سهولة.

كذلك ما يجعل الـ object رائع هو إمكانية تمريرة وكل ما بدأخلة لـ function كعنصر واحد بدلاً من تمرير كل متغير على حدة وفي نفس الوقت نستطيع التعامل مع كل هذه البيانات وإرجاع كل هذه المعلومات بإستخدام return.

فكما نعرف بأن return لا تقوم الا بإرجاع قيمة واحدة فقط و بإستخدام الـ Object سنستطيع إرجاع العديد من القيم مغلفة تحت إسم واحد.

ما الفرق بين الـ Arrays و الـ Objects

ربما تتسأل ما الفرق بينها طالما وأنها تقوم بتخزين عدد من البيانات لذا لنقوم بتوضيح بعض النقاط.

الـ Array نوع خاص من الـ object:
لنأكد من ذلك قم بتعريف المصفوفة التالية وطباعة نوعها بإستخدام الدالة typeof كالتالي: 

var array = [];

console.log(typeof (array));

النتيجة:

الـ arrays تستخدم الأرقام كـ index إلقي نظرة على الصورة التالية:

بمعنى أنك إذا اردت أن تصل لمحتوى الخزانة السادسة ستقوم بالتالي:

locker[5];

الـ Objects تستخدم الأسماء كـ Key إلقي نظرة على الصورة التالية:

فإذا اردت الوصول للخزانة السادسة ستقوم بإستخدام إسمها كالتالي:

locker["employees"];

او هكذا

locker.employees;

متى تستخدم الـ arrays ومتى تستخدم الـ objects:

  • عندما تريد تخزين قائمة من البيانات التي هي من نفس النوع قم بإستخدام Array
var colors = ["red", "white", "black"];
  • عندما تريد تخزين بيانات كخصائص يجمعها عامل مشترك ولها انواع مختلفة قوم بإستخدام object
var student = {
    id : 12345,
    name : "Mohammed",
    registered : true
};

لا نستطيع نقاش ماذا تختار بشكل مفصل فلن نتوقف في سرد التفضيلات ولكن النقطتين التي طرحتهم اعلاة بشكل عام وعند بحثك ودراستك للـ Objects ستعرف متى تستخدمها ومتى لا تستخدمها.

طباعة عناصر الـ Object و الـ for-in-loop

طباعة الـ array

كما ذكرنا أعلاة فالمصفوفات عناصرها يتم إعطائها index رقمي يبدأ من صفر 0 وينتهي حسب طول المصفوفة فإذا كان لدينا المصفوفة التالية:

var colors = ["red", "white", "black"];

ونريد طباعتها فسوف نستخدم الحلقات لنأخذ for كالتالي:

for ( var i = 0 ; i < colors.length ; i++){

    console.log( colors[i] );

}

لاحظ كيف أننا نستطيع المرور على جميع عناصر المصفوفة وطباعتها بإستخدام العداد i الذي يبدأ بالعد من صفر. في حال تواجهك مصاعب في الحلقات قم بمراجعة الدرس الخاص بها .

طباعة الـ object

كما ذكرنا الـ Object لا يعتمد على قيمة رقمية كـ index إنما يعتمد على قيمة نصية تسمى الـ Key او الـ Property.

هناك حلقة loop تسمى for-in-loop تستخدم للقيام بذلك:

لاحظ تركيب الحلقة حيث نقوم بتعريف متغير بإي إسم انا قمت بتسميته key تم نكتب in ونتبعها بإسم الـ object كما قمنا بتعريفة سابقاً student بعد ذلك نقوم بطباعته بإستخدام الكود student[key] حيث مررنا المتغير الخاص بنا key بداخل الأقواس المربعة للـ object كالتالي:

var student = {
    id : 12345,
    name : "Mohammed",
    registered : true
};

for( var key in student ){

    console.log( student[key]);

}

لا يمكنك إستخدام الطريقة student.key في هذه الحلقة فتذكر ذلك.

عمل Arrays of objects

نستطيع عمل أكثر من object بدأخل Array لنقوم بعمل تطبيق.

سأقوم بعمل مصفوفة بإسم products كالتالي:

var products = [];

الأن سأقوم بعمل منتجات بدأخل هذه المصفوفة بالشكل التالي:

var products = [

    {
      name : "iphone",
      price : 1200,
      colors : ["red", "black", "white"],
      available : true
    }

];

في الكود اعلاة قمت بعمل مصفوفة وبدأخلها object لنقوم الأن بإضافة object اخر:

var products = [

    {
      name : "iphone",
      price : 1200,
      colors : ["red", "black", "white"],
      available : true
    },

    {
      name : "canon camera",
      price : 1000,
      colors : ["black", "gray"],
      available : false
    }

];

الأن بكل بساطة يمكنك الوصول لاي عنصر بدأخل أي object داخل هذه الـ array لنبدأ حبة حبة كالتالي:

كأي مصفوفة عادية لدينا عنصرين objects بداخل هذه المصفوفة كالتالي:

بما يعني أنه اذا اردنا الوصول للـ object الأول فنقوم بتحديد الـ Index الخاص بالمصفوفة كالتالي:

products[0];

الأن لنفترض بأنني أريد أن أطبع إسم المنتج الذي في الـ object الأول, سنقوم كما شرحنا في السابق بإستخدام احد الطريقتين:

إما هكذا:

products[0].name;

الـ JSON و الـ Ajax

بما أن الـ Objects تتيح لنا هذه الطريقة الرائعة لتخزين العديد من البيانات وتسمح لنا بالوصول إليها بهذه السهولة فقد تم إشتقاق منها صيغة كلغة لتبادل البيانات على الإنترنت بإسم (JSON) حيث تقوم بتمثيل البيانات بطريقة مرتبة كنصوص ومن ثم نستطيع تبادل هذه البيانات مع السيرفر او مع لغات أخرى فالـ JSON نستطيع القول بأنها وسيط بين اللغات وتستطيع اللغات الأخرى فهمها وإستخلاص البيانات منها.

الـ JSON تستخدم بشكل كبير مع تقنية Ajax إن كان قد سبق وان سمعت بها حيث تقوم الـ Ajax بعمل تبادل للبيانات بين السيرفر والمتصفح ما يجعل الموقع لدية القدرة على جلب بيانات دون الحاجة لعمل تحديث للصفحة على سبيل المثال:

هل لاحظت التقنية المستخدمة في الموقع عند كتابة أحدهم لتعليق يظهر بأنه يقوم بكتابة تعليق ومن ثم يتم إضافة التعليق للصفحة دون أن يطلب منكم ان تقوم بتحديث الصفحة او كيف أن التغييرات تتم مباشرة عند تعديلك لموضوع بمجرد ضغط حفظ دون إعادة تحميل الصفحة

مثال لمستند JSON:

myObj = {
    "name":"John",
    "age":30,
    "cars": [
        "car1":"Ford",
        "car2":"BMW",
        "car3":"Fiat"
     ]
 }
